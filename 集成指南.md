# 消控事件规则配置系统集成指南

本文档详细说明如何将消控事件规则配置系统集成到现有的 RCS（RobotControlSystem）WPF 项目中。

---

## 文件清单

需要将以下文件复制到项目中对应的位置：

### 1. Models 目录
- `Models/EventRule.cs` → 项目中的 `RobotControlSystem/Models/EventRule.cs`

### 2. Services 目录
- `Services/EventRuleStore.cs` → 项目中的 `RobotControlSystem/Services/EventRuleStore.cs`
- `Services/EventMatchEngine.cs` → 项目中的 `RobotControlSystem/Services/EventMatchEngine.cs`
- `Services/EventProcessor.cs` → 项目中的 `RobotControlSystem/Services/EventProcessor.cs`

### 3. Editors 目录
- `Editors/EventRuleDesignerWindow.xaml` → 项目中的 `RobotControlSystem/Editors/EventRuleDesignerWindow.xaml`
- `Editors/EventRuleDesignerWindow.xaml.cs` → 项目中的 `RobotControlSystem/Editors/EventRuleDesignerWindow.xaml.cs`

## 需要修改的现有文件

### 1. App.xaml.cs

**目标**：初始化 EventProcessor，移除硬编码的事件处理逻辑

**修改步骤**：

1. 添加命名空间引用：
```csharp
using RobotControlSystem.Services;
```

2. 在 `OnStartup` 方法中初始化 EventProcessor：
```csharp
protected override void OnStartup(StartupEventArgs e)
{
    base.OnStartup(e);

    // 初始化消控事件处理器
    EventProcessor.Instance.Initialize();
    System.Diagnostics.Debug.WriteLine("[App] 消控事件处理器已初始化");
}
```

3. 如果存在 `OnExit` 方法，添加清理逻辑：
```csharp
protected override void OnExit(ExitEventArgs e)
{
    // 清理事件处理器
    EventProcessor.Instance.Shutdown();
    base.OnExit(e);
}
```

4. **移除或修改**原有的 `OnUserDeviceStatusChanged` 方法中的硬编码火警处理逻辑：
   - 原逻辑可能直接在事件回调中判断 `FireAlarm` 类型并执行处理
   - 现在应该由 EventProcessor 自动处理
   - 如果需要响应规则匹配，订阅 `EventProcessor.Instance.RuleMatched` 事件

**示例改造前**（假设原代码）：
```csharp
private void OnUserDeviceStatusChanged(object? sender, DeviceEventArgs e)
{
    if (e.Status == DeviceStatus.FireAlarm)
    {
        // 硬编码的火警处理逻辑，可能阻塞UI
        HandleFireAlarm(e.Message);
    }
}
```

**示例改造后**（订阅事件处理器）：
```csharp
// 在适当的初始化位置（如构造函数或 OnStartup）
EventProcessor.Instance.RuleMatched += OnRuleMatched;

// 事件匹配回调方法
private void OnRuleMatched(EventRule rule, DeviceEventArgs e)
{
    // 根据匹配的规则执行对应的配方
    System.Diagnostics.Debug.WriteLine($"执行配方: {rule.RecipeName}");
    
    // 调用配方执行逻辑
    ExecuteRecipe(rule.RecipeName, e);
}
```

**目标**：实现按钮点击事件，初始化 EventProcessor

**修改步骤**：

```csharp
using RobotControlSystem.Services;
```

2. 添加按钮点击事件处理方法：

```csharp
private void BtnEventRuleDesigner_Click(object sender, RoutedEventArgs e)

```

3. 在 `Window_Loaded` 或类似方法中确保 EventProcessor 已初始化：
```csharp
private void Window_Loaded(object sender, RoutedEventArgs e)
{
    // 确保事件处理器已初始化（如果在 App.xaml.cs 中未初始化）

    {
        EventProcessor.Instance.Initialize();
    }

    // 其他初始化逻辑...

```

4. **改造原有的火警处理方法** `OnFireAlarmReceived`：

**改造前**（假设原代码）：
```csharp
private void OnFireAlarmReceived(object? sender, DeviceEventArgs e)
{
    // 直接在UI线程处理，可能阻塞
    ParseAndAddData(e.Message);
    UpdateUI(e);
}
```

**改造后**（使用 EventProcessor）：
```csharp
// 移除直接的事件订阅，由 EventProcessor 统一处理
// 如果需要在规则匹配后执行特定逻辑，订阅 RuleMatched 事件：

private void OnRuleMatched(EventRule rule, DeviceEventArgs e)
{
    // 根据规则类型处理
    switch (rule.EventType)
    {
        case EventType.火警报警器:
            // 异步处理，避免阻塞UI
            _ = Task.Run(() => ParseAndAddData(e.Message ?? ""));
            UpdateUI(e);
            break;
        
        case EventType.远端命令:
            // 处理远端命令
            HandleRemoteCommand(rule.RecipeName, e);
            break;
            
        default:
            // 其他类型
            break;
    }
}
```

### 4. FireMonitorWindow.xaml.cs

**目标**：改造设备状态变化事件处理，避免UI阻塞

**修改步骤**：

1. 找到 `OnDeviceStatusChanged` 方法（或类似的事件处理方法）

2. **改造前**（假设原代码）：
```csharp
private void OnDeviceStatusChanged(object? sender, DeviceEventArgs e)
{
    // 直接在事件回调中处理，阻塞UI线程
    if (e.Status == DeviceStatus.FireAlarm)
    {
        ParseAndAddData(e.Message);  // 可能耗时
        UpdateLog(e.Message);
    }
}
```

3. **改造后**：
```csharp
private void OnDeviceStatusChanged(object? sender, DeviceEventArgs e)
{
    // 方式1：移除此方法，由 EventProcessor 统一处理
    // 如果已经在 App.xaml.cs 中初始化了 EventProcessor，可以删除此方法

    // 方式2：如果需要保留此方法用于其他逻辑，改为异步处理
    _ = Task.Run(() =>
    {
        try
        {
            // 模拟耗时处理
            System.Threading.Thread.Sleep(100); // 模拟耗时操作
            
            // 通过 Dispatcher 更新UI
            this.Dispatcher.Invoke(() =>
            {
                UpdateLog(e.Message);
            });
        }
        catch (Exception ex)
        {
            System.Diagnostics.Debug.WriteLine($"处理异常: {ex.Message}");
        }
    });
}
```

4. 如果此窗口需要响应规则匹配事件：
```csharp
// 在窗口构造函数或 Loaded 事件中订阅
EventProcessor.Instance.RuleMatched += OnFireAlarmMatched;

// 匹配回调
private void OnFireAlarmMatched(EventRule rule, DeviceEventArgs e)
{
    // 只处理火警类型
    if (rule.EventType == EventType.火警报警器)
    {
        // 异步处理数据解析
        _ = Task.Run(() => 
        {
            var data = ParseAndAddData(e.Message ?? "");
            
            // 更新UI
            this.Dispatcher.Invoke(() =>
            {
                UpdateFireMonitorUI(data);
            });
        });
    }
}

// 在窗口关闭时取消订阅
private void Window_Closing(object sender, System.ComponentModel.CancelEventArgs e)
{
    EventProcessor.Instance.RuleMatched -= OnFireAlarmMatched;
}
```

## 常见问题

### Q1: 编译错误："找不到类型或命名空间名 'EventType'"

**A**: 确保 `Models/EventRule.cs` 文件已正确添加到项目，且命名空间为 `RobotControlSystem.Models`。

### Q2: 运行时 EventRules.json 文件未生成

**A**: 检查程序是否有写入应用程序根目录的权限。如果在发布版本中运行，可能需要管理员权限或调整文件路径。

### Q3: 点击"保存"按钮后规则未保存

**A**: 检查 `EventRuleStore.cs` 中的文件路径是否正确，并确保程序有写入权限。

### Q4: 界面仍然卡死

**A**: 确保：
1. 已移除原有的事件处理方法中的同步处理逻辑
2. 所有耗时操作都在 `Task.Run` 中执行
3. UI 更新通过 `Dispatcher.Invoke` 完成

### Q5: 规则匹配不工作

**A**: 检查：
1. 规则是否已启用（IsEnabled = true）
2. 事件类型和消息内容是否匹配
3. EventProcessor 是否已初始化
4. 查看调试输出中的匹配日志

## 扩展建议

### 1. 数据库集成

如果需要从数据库加载配方名称列表，修改 `EventRuleDesignerWindow.xaml.cs` 中的 `LoadRecipeNames` 方法：

```csharp
private void LoadRecipeNames()
{
    // 示例：从 SQLite 数据库加载
    using (var connection = new SQLiteConnection("YourConnectionString"))
    {
        connection.Open();
        var command = new SQLiteCommand("SELECT RecipeName FROM Recipes", connection);
        var reader = command.ExecuteReader();
        
        RecipeNames = new List<string>();
        while (reader.Read())
        {
            RecipeNames.Add(reader.GetString(0));
        }
    }
}
```

### 2. 规则导入导出

添加规则导入导出功能，支持JSON文件的导入导出：

```csharp
// 导出规则
private void ExportRules()
{
    var rules = EventRuleStore.Instance.GetAllRules();
    var json = JsonConvert.SerializeObject(rules, Formatting.Indented);
    File.WriteAllText("ExportedRules.json", json);
}

// 导入规则
private void ImportRules(string filePath)
{
    var json = File.ReadAllText(filePath);
    var rules = JsonConvert.DeserializeObject<List<EventRule>>(json);
    // 合并或替换现有规则
}
```

### 3. 规则版本控制

如果需要记录规则变更历史，可以：
- 在 `EventRule` 模型中添加 `CreatedAt` 和 `UpdatedAt` 字段
- 创建 `RuleHistory` 表存储规则变更历史
- 每次保存规则时记录历史

### 4. 高级匹配条件

扩展 `MatchCondition` 枚举，添加更多匹配方式：
```csharp

{

    完全等于,
    开头是,
    结尾是,
    正则匹配,
    长度大于,      // 新增
    长度小于,      // 新增
    JSON字段匹配    // 新增
}
```
