### 1. 原始数据接收日志
```
[原始接收] 4040CA00... 字节数=30
```
记录每次TCP接收的原始数据。

### 2. 解析包信息日志
```
[解析包] 长度=30 应用数据长度=0 命令=0x02
```
记录解析出的包长度、应用数据长度、命令类型。

### 3. 结束符错误日志
```

```

### 4. 校验失败日志
```

```
如果校验和不匹配，记录期望值和计算值。

### 5. 业务数据包日志
```
[业务数据] 命令=0x02 设备=C8-00-00-00-00-00 应用数据=...
```

## 日志分析

### 如果只看到心跳包：
```
[原始接收] 4040... 字节数=30
[解析包] 长度=30 应用数据长度=0 命令=0x02
[收到请求] ...
[发送回复] ...
```
说明用传设备还没发送火警数据，可能是：
- 用传设备等待心跳成功后才发送
- 用传设备配置问题

### 如果看到校验失败：
```
[原始接收] 4040... 字节数=78
[解析包] 长度=78 应用数据长度=48 命令=0x02
[校验失败] expected=0x7B calc=0x5A ...
```
说明用传设备发送了火警数据，但校验和计算方式不同。

### 如果看到业务数据：
```
[解析包] 长度=78 应用数据长度=48 命令=0x02
[业务数据] 命令=0x02 设备=... 应用数据=...
```
说明火警数据已成功接收并解析。

## 预期的火警数据包结构

根据测试数据：
- 总长度: 76字节（心跳回复30字节 + 应用数据48字节 - CRC重叠2字节）
- 应用数据长度: 48字节（0x0030）
- 命令: 0x02（部件状态）

应用数据内容：
- 类型标识: 2字节
- 信息数量: 2字节
- 信息体: 44字节（部件类型、地址、状态、描述等）
